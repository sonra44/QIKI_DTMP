# ORION V top-zone principles

## Purpose

This reference keeps the visual redesign focused on operator work instead of decorative churn.

## Principles

1. Name every strip.
   If the operator cannot answer "what is this band for?" in one glance, the strip is underdesigned.

2. One strip, one job.
   - Header: global state
   - Navigation: screen switching and universal actions
   - Core systems: fast health scan
   - Alerts: only urgent attention
   - Command line: explicit operator input
   - Quick actions: local cockpit jumps / QIKI confirms

3. Use titles to carry structure.
   Border titles and subtitles should absorb explanation so button labels can stay short.

4. Do not overload bilingual text.
   `RU/EN` is required, but bilingual does not mean verbose.

5. Prefer calm hierarchy.
   Use one strong anchor at the top and softer secondary strips below.
