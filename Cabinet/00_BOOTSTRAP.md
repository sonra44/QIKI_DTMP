# Старт сессии (QIKI_DTMP)

Минимальный порядок, чтобы не «уехать в сторону».

## 1) Поднять контекст (обязательно)

- Прочитать: `~/MEMORI/OPERATIVE_CORE_QIKI_DTMP.md`
- Прочитать: `~/MEMORI/ACTIVE_TASKS_QIKI_DTMP.md`
- Поднять память по тегам (детерминированно): `STATUS`, `TODO_NEXT`, `DECISIONS`, `TASKS`, `DEV_METHODS`

## 2) Протокол «60 секунд» перед крупным шагом

1) Цель в 1 строку + критерий успеха
2) Инварианты:
   - Docker-first
   - no-mocks (если данных нет — `N/A/—`)
   - no-duplicates/no-v2 (не плодим второй канон)
3) Состояние репо: `git status --porcelain`
4) Точка входа: определить фактический entrypoint (compose/скрипт)
5) Мини-smoke под цель (1 команда, 30–60 секунд)

## 3) Фиксация в конце «петли»

- Сохранить `STATUS` + `TODO_NEXT` (+ `DECISIONS`, если появилось долгоживущее решение)
- Сразу сделать `recall_by_tags` и показать ID записей (пруф)
