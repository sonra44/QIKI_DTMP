# Методика и принципы разработки (QIKI_DTMP)

## 0) Почему это нужно

Цель — убрать хаос: одна каноническая доска, понятный DoD, и запрет на параллельные “источники правды”.

## 1) Границы (scope guard)

- Если задача не лежит в `~/MEMORI/ACTIVE_TASKS_QIKI_DTMP.md`, то она либо:
  - добавляется туда (явно), либо
  - считается «вне спринта» и откладывается.

## 2) Definition of Done (минимальный DoD)

Для изменения “считается сделано”, когда:
- Docker-first проверка пройдена (команда и вывод зафиксированы).
- Если это ORION UI — чеклист в `docs/design/operator_console/ORION_OS_VALIDATION_CHECKLIST.md` закрыт ✅/❌ по релевантным пунктам.
- Документация обновлена по `DOCUMENTATION_UPDATE_PROTOCOL.md` (если менялось поведение/хоткеи/инварианты).
- Репозиторий в чистом состоянии (изменения либо закоммичены, либо сознательно откатаны).

## 3) Анти-дубли

- Не создаём `*_v2`, не копируем канонические файлы в новые папки.
- Вместо копирования: индекс/ссылки + короткие “why/how” заметки.

## 4) Как не прыгать между задачами

- Один “active objective” за раз (из блока **A) Now** канонической доски).
- Любой сайд-квест → отдельная запись в `TODO_NEXT` с явным “почему сейчас” и “что блокирует”.
