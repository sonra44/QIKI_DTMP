# Кабинет (QIKI_DTMP)

Цель: единая «точка входа» по проекту — **как работаем**, **где канон**, **как фиксируем прогресс**, чтобы не прыгать между задачами и не плодить дубли.

## Канон (source of truth)

1) Каноническая доска задач (верхний уровень): `~/MEMORI/ACTIVE_TASKS_QIKI_DTMP.md`
2) Операционное ядро (инварианты, протокол): `~/MEMORI/OPERATIVE_CORE_QIKI_DTMP.md`
3) ORION Shell OS ручная валидация (✅/❌): `docs/design/operator_console/ORION_OS_VALIDATION_CHECKLIST.md`
4) Обновление документации при изменениях: `DOCUMENTATION_UPDATE_PROTOCOL.md`

> Примечание: `~/MEMORI/*` — внешние для репозитория артефакты. В Кабинете мы **не копируем канон**, а держим ссылки/правила, чтобы не создавать второй источник правды.

## Структура Кабинета

- `QIKI_DTMP/Cabinet/00_BOOTSTRAP.md` — старт сессии и «60 секунд» протокол
- `QIKI_DTMP/Cabinet/01_WORKFLOW.md` — методика разработки и правила фиксации (DoD/гейт, без дублей)
- `QIKI_DTMP/Cabinet/02_TASKS.md` — как вести задачи “Сделано/Не сделано” (шаблоны + привязка к канону)
- `QIKI_DTMP/Cabinet/03_INDEX.md` — индекс “где что лежит”
- `QIKI_DTMP/Cabinet/Reports/` — отчёты по фактам: техдолг / дубли / баги / риски
