# HOT_TEST_STATESTORE.SH — анализ по методу «Задачи»

## Назначение файла
Скрипт запускает комплексную HOT TEST проверку архитектуры StateStore: от проверки окружения до генерации отчёта о состоянии системы.

## Основные задачи
- [ ] `check_environment` — убедиться, что выполняем в корне проекта и установлены зависимости (`pytest`, `asyncio`, `psutil`).
- [ ] `syntax_check` — компиляция ключевых модулей (`types.py`, `store.py`, `conv.py`).
- [ ] `import_check` — проверка импортов основных классов и функций StateStore.
- [ ] `functional_test` — минимальный сценарий использования стора и конвертеров.
- [ ] `run_unit_tests` — запуск unit‑тестов `test_types.py`, `test_store.py`, `test_conv.py`.
- [ ] `run_integration_tests` — запуск интеграционных тестов `test_integration.py`.
- [ ] `run_stress_tests` — сокращённые стресс‑тесты (`concurrent_writers_stress`, `high_volume_sets_and_gets`).
- [ ] `performance_test` — измерение скорости операций `set/get`.
- [ ] `compatibility_test` — проверка совместимости DTO с protobuf типами.
- [ ] `memory_leak_test` — обнаружение потенциальных утечек памяти в сценариях с подписчиками.
- [ ] `generate_report` — формирование итогового отчёта `HOT_TEST_REPORT_*.md`.
- [ ] Обработка аргументов CLI (`--unit-only`, `--stress-only`, `--quick`, `--help`).

## Наблюдения и рекомендации
- Скрипт ориентирован на быстрый локальный запуск; длительные проверки (`stress`, `perf`) выполняются в облегчённом режиме.
- Для стабильной работы требуется предустановленный protobuf‑код в каталоге `generated/`.
- При необходимости расширения тестов удобно добавлять новые функции по аналогии с существующими.
