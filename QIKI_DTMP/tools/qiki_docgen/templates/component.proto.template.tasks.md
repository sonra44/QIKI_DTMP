# Анализ component.proto.template

## Задачи
1. Задать имя пакета и префикс компонента (`component_name`).
2. Переименовать сообщение `{{ component_proto_type }}` в конкретный тип компонента.
3. Дополнить описание и набор полей:
   - заполнить поле `description`;
   - при необходимости добавить скалярные, перечислимые (enum), повторяющиеся (`repeated`), `oneof` и `map` поля.
4. При необходимости расширить протокол: дополнительные импорты, сообщения или перечисления.
5. Проверить наличие новой строки в конце файла.
