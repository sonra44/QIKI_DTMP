# Отчёт по minimal_component.proto.template

## Вход и цель
- [Факт] Изучить упрощённый шаблон протокола компонента.

## Сбор контекста
- [Факт] Синтаксис `proto3`, пакет `qiki.{{ component_name | lower }}`.
- [Факт] Импортирует `common_types.proto`.
- [Гипотеза] Используется для минимальных примеров без дополнительных полей.

## Локализация артефакта
- [Факт] `tools/qiki_docgen/templates/minimal_component.proto.template`.
- [Факт] Применяется в ситуациях, когда требуются только `id` и `name`.

## Фактический разбор
- [Факт] Сообщение `{{ component_proto_type }}` содержит поля `id` и `name`.
- [Гипотеза] Дополнительные поля добавляет пользователь при расширении.

## Роль в системе и связи
- [Факт] Обеспечивает минимально жизнеспособный контракт для сервиса.
- [Гипотеза] Может использоваться в прототипах и тестовых окружениях.

## Несоответствия и риски
- [Гипотеза][Med] Отсутствие `created_at` и других метаданных усложняет аудит.

## Мини-патчи (safe-fix)
- [Патч] Добавить необязательное поле `string description = 3;`.

## Рефактор-скетч (по желанию)
```proto
message {{ component_proto_type }} {
  qiki.common.UUID id = 1;
  string name = 2;
  // [Патч] string description = 3;
}
```

## Примеры использования
```bash
# [Факт] Компиляция минимального протокола
protoc minimal_component.proto
```
```python
# [Гипотеза] Создание объекта
```

## Тест-хуки/чек-лист
- [Факт] Проверить уникальность `id`.
- [Гипотеза] В интеграционных тестах убедиться, что расширенные поля игнорируются корректно.

## Вывод
- [Факт] Минимальный шаблон подходит для простых случаев.
- [Патч] При необходимости расширить метаданными.
