# ARTIFACT: ORION V navigation/action logic audit

Дата: 2026-03-06
Статус: complete

## Зачем

Пользовательский запрос сместил приоритет с косметики на завершённость и понятность:
- каждое меню должно иметь ясный смысл;
- каждая кнопка должна быть уместна в своём контексте;
- интерфейс не должен держать бесполезную или дублирующую информацию.

## Найденные дефекты

1. `action_bar` нарушал канон `No UI abbreviations`.
- Были строки `Инцид.`, `Подтв.`, `Впер.`, `Поток`, `Статус`.
- Это делало верхнюю навигацию похожей на служебную панель, а не на операторский интерфейс.

2. Глобальные действия висели без контекста.
- Кнопки для инцидентов и перелистывания были видны всегда.
- Вне `F1/F3` или `F3/F6` часть из них не имела ясного немедленного смысла.

3. `status_bars` дублировали смысл.
- В значении строки повторялось имя метрики.
- Дополнительно вшивалась короткая подсказка `click→F2/F3`, что давало шум вместо пользы.

## Что изменено

### 1. Action bar

Файл:
- `src/qiki/services/operator_console/orion_v/widgets/action_bar.py`

Сделано:
- убраны сокращения;
- названия экранов и действий даны явно;
- действия для инцидентов и страниц теперь показываются только там, где они уместны;
- подтверждение инцидента остаётся запрещённым в replay mode.

Новый принцип:
- экраны `F1-F7` всегда видны;
- контекстные действия появляются только при наличии операторского смысла.

### 2. App wiring

Файл:
- `src/qiki/services/operator_console/orion_v/app.py`

Сделано:
- `OrionVActionBar.set_state(...)` теперь получает флаги контекста;
- `incident`-действия активны только на `F1/F3`;
- `page`-действия активны только на `F3/F6`;
- subtitle навигации теперь объясняет этот принцип прямо.

### 3. Status bars

Файл:
- `src/qiki/services/operator_console/orion_v/widgets/status_bars.py`

Сделано:
- кнопки метрик переименованы в более ясные RU/EN ярлыки;
- title объясняет, что кнопка слева открывает детали;
- значение строки больше не дублирует имя метрики;
- убраны служебные короткие подсказки `click→F2/F3`.

## Документация

Файл:
- `docs/design/operator_console/CANONICAL_SPEC_ORION_QIKI.md`

Добавлен инвариант:
- глобальная навигация может быть видимой всегда,
- но screen-specific actions должны скрываться или отключаться вне релевантного контекста.

## Честный остаток

После этого прохода интерфейс стал логичнее именно по действиям и меню.
Но два смысловых блока всё ещё перегружены и требуют отдельного прохода:
- `QIKI`
- `Движение и навигация`

Следующий сильный логический шаг в UI — не переименование рамок, а сокращение и структурирование этих двух блоков по правилу:
- сначала решение/состояние,
- затем причина,
- затем следующий шаг.
