# Назначение
Этот агент предназначен для исправления ошибок и падений тестов в текущем репозитории.

# Контекст
- Репозиторий построен на Python 3.12, используется фреймворк FastAPI, асинхронный код на asyncio.
- Исходный код расположен в каталоге `src/`, тесты — в каталоге `tests/`.
- Следуем стандарту PEP 8; для проверки стиля запускаем линтер **Ruff** (`ruff check src tests`) и исправляем все нарушения.
- Для измерения качества кода используем **Pylint**. Итоговая оценка не должна быть ниже 8/10 (`pylint src`).
- Тесты выполняются PyTest (`pytest --cov=src`) с плагином coverage; покрытие должно быть не менее 80 %.
- Статические модели в коде используют Pydantic v2; работаем в строгом режиме и не используем устаревшие API.

# Задача
1. Запусти тесты PyTest и приложи в контекст их вывод. Выяви фейлящие тесты и расскажи, какие причины ошибок нашёл.
2. Анализируя код, предложи минимальные изменения, чтобы исправить ошибки, не нарушая архитектуру и соглашения проекта.
3. Перед каждым изменением запускай Ruff и Pylint, исправь замечания. Проверь, что итоговая оценка Pylint ≥ 8/10.
4. Напиши исправления кода и покажи мне diff‑панель. Не меняй глобально структуру проекта.
5. Запусти тесты ещё раз. Добейся, чтобы все тесты проходили и покрытие было ≥ 80 %.

# Ожидаемый результат
- Все тесты проходят без ошибок.
- Исправленные файлы соответствуют PEP 8 и имеют качество кода по Pylint не ниже 8/10.
- Покрытие кода ≥ 80 %.
- Агент подготовил чёткое сообщение коммита на английском с описанием исправлений.
