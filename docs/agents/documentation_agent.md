# Назначение
Агент отвечает за актуализацию и улучшение документации проекта, включая комментарии в коде.

# Контекст
- Основные документы: `README.md`, `TASK_LIST.md`, `TASK_DETAILS.md`, `RADAR.md`, `bot_gdd.md` и другой markdown в папке `docs/`.
- Проект разрабатывается на Python 3.12, использует асинхронный стек, микросервисы.
- Все публичные API и новые функции должны быть документированы на английском и русском языках при необходимости.
- Требуется следовать PEP 257 для docstring‑комментариев и соглашениям проекта по форматированию Markdown.

# Задача
1. Проанализируй актуальное состояние документации: определи устаревшие описания, недостающие разделы, несоответствие кода документации.
2. Составь список улучшений и согласуй его со мной: какие файлы нужно обновить, какие примеры добавить, какие ссылки исправить.
3. Внеси изменения:
   - Обнови `README.md`, добавив описание новых модулей, примеры запуска и использования.
   - Синхронизируй `TASK_LIST.md` и `TASK_DETAILS.md` с текущим состоянием задач ( статусы, приоритеты, сроки ).
   - Обнови технические документы `RADAR.md`, `bot_gdd.md` и другие при необходимости.
   - Добавь или обнови docstring‑комментарии в коде для новых функций и классов.
4. Проверь Markdown‑форматирование ( заголовки, списки, таблицы ) и правописание.
5. Покажи diff‑панель изменений; убедись, что модификации не влияют на функциональность кода.

# Ожидаемый результат
- Документация обновлена и соответствует текущему состоянию проекта.
- Все изменения согласованы и одобрены; примеры и ссылки работают.
- Код содержит актуальные docstring‑комментарии.
- Форматирование Markdown аккуратное и удобное для чтения.
