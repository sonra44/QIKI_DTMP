# Анализ файла: `protos/common_types.proto`

**Версия документации:** 1.0  
**Дата анализа:** 2025-09-03

---

## 1. Назначение и Архитектурная Роль

Этот файл определяет **фундаментальные, многократно используемые типы данных**, которые являются общими для различных сервисов и сообщений в платформе QIKI_DTMP. Он служит своего рода **"стандартной библиотекой" Protobuf-типов** для проекта.

**Архитектурная роль:** `common_types.proto` формирует **базовый слой иерархии Protobuf-контрактов**. Его наличие обеспечивает:
-   **Повторное использование:** Предотвращает дублирование общих определений в разных `.proto` файлах.
-   **Консистентность:** Гарантирует, что базовые понятия, такие как UUID, 3D-векторы и единицы измерения, используются единообразно по всей системе.
-   **Типобезопасность:** Обеспечивает строгую типизацию для общих концепций, что снижает вероятность ошибок при обмене данными.

---

## 2. Детальный Разбор Определений

### 2.1. Сообщение `UUID`

-   **Назначение:** Представляет универсальный уникальный идентификатор (UUID).
-   **Поле:** `string value = 1;`.
-   **Значимость:** Используется повсеместно для уникальной идентификации объектов, сообщений, снимков состояния и т.д. Это критически важно для трассировки и корреляции событий в распределенной системе.

### 2.2. Сообщение `Vector3`

-   **Назначение:** Представляет 3D-вектор.
-   **Поля:** `float x = 1;`, `float y = 2;`, `float z = 3;`.
-   **Значимость:** Используется для представления позиций, направлений, скоростей и других векторных величин. Комментарий в файле (`Coordinate system: [Specify your coordinate system here, e.g., ROS REP 103, right-handed, Z-up]`) подчеркивает важность четкого определения системы координат для физических симуляций.

### 2.3. Перечисления (Enums)

-   **`SensorType`**: Определяет стандартизированные типы сенсоров.
    -   **Значения:** `LIDAR`, `IMU`, `CAMERA`, `GPS`, `THERMAL`.
    -   **Значимость:** Позволяет однозначно идентифицировать тип сенсора, что важно для обработки данных и конфигурации.

-   **`ActuatorType`**: Определяет стандартизированные типы исполнительных механизмов.
    -   **Значения:** `WHEEL_MOTOR`, `SERVO`, `GRIPPER`, `ARM`.
    -   **Значимость:** Аналогично `SensorType`, обеспечивает единообразие в идентификации актуаторов.

-   **`Unit`**: Определяет стандартизированные единицы измерения.
    -   **Значения:** `METERS`, `DEGREES`, `PERCENT`, `VOLTS`, `AMPS`, `WATTS`, `MILLISECONDS`, `KELVIN`, `BAR`.
    -   **Значимость:** Наличие таких единиц, как `KELVIN` (температура) и `BAR` (давление), а также `METERS` и `DEGREES`, явно указывает на **направленность проекта на физические симуляции и робототехнику/космические аппараты**.

## 3. Взаимодействие с Другими `.proto` Файлами

`common_types.proto` импортируется и используется практически всеми другими `.proto` файлами в проекте:

-   `sensor_raw_in.proto`
-   `actuator_raw_out.proto`
-   `bios_status.proto`
-   `fsm_state.proto`
-   `proposal.proto`
-   `q_sim_api.proto`

Это подтверждает его роль как фундаментального слоя для определения общих типов данных.
