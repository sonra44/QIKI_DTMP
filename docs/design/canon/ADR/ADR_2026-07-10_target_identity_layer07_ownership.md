# ADR: Идентичность целей — владелец Layer-07 fusion = мозг; мост = транспорт; транспондер-якорь

**Status:** ACCEPTED (design-only; K2-код после пакета playable)
**Date:** 2026-07-10
**Decision owner:** оператор + CLI-агент (карта `docs/dev/AUDIT_2026-07-10_STRUCTURE.md`, N9/S1/N7/N8)

## Context

- Канон сам зафиксировал дыру:
  `docs/design/sensor_observation/SENSOR_OBS_0001_OBJECTIVE_TO_SENSOR_BOUNDARY.md`
  относит `radar-track fusion`, `operator target inference`,
  `sensor trust decision` к «later Layer-07 and Layer-08 work»
  (Non-goals). Владелец фьюжна канонически НЕ назначен — три
  независимых матчинга в коде это следствие отложенного канон-решения,
  а не «плохой код» (N9). (Прим.: doc-часть патча SENSOR_OBS_0001 не
  доехала до репо вместе с кодом — восстановлена в `docs/` из
  `ПАЧИ/999999/…sensor_obs_0001…patch` тем же коммитом, что этот фикс.)
- Код-правда (подтверждено file:line на main=9486295, спот-чек 70a5d3f):
  - мозг: `q_core_agent/core/world_model.py:243` — пороги ассоциации
    2500 м / 12° / 8° / 250 м/с; треки мозга питают FSM/гварды;
  - мост: `faststream_bridge/radar_track_store.py:73-74` — 12 м евклид /
    15 м/с, плюс `radar_handlers._select_best_track` схлопывает кадр в
    ОДИН трек — консоль ORION V видит мостовую идентичность;
  - N7: `q_core_agent/core/ship_fsm_handler.py:414-443`
    `_get_best_station_track` читает proto `radar_track` напрямую из
    сенсорных чтений — четвёртый путь идентичности, мимо WorldModel;
  - N8: intents уже платит налог идентичности — двухключевой carry
    `qcore_track_id + public_track_id` с fallback-резолвом (19 упоминаний
    `public_track_id` в `qiki_orion_intents_service.py`; таблицы
    трансляции НЕТ — оба ключа ищутся в собственном world_snapshot
    мозга №2, public-ветка в кросс-доменном случае практически мертва,
    т.к. UUID моста и мозга генерятся независимо).
- Следствие: оператор и мозг видят РАЗНЫЕ цели (track_id несравнимы) —
  противоречит духу P2 «радар не лжёт» и P3 (наблюдаемость решений);
  этап 9b (интент по треку сцены) без решения строится на песке.
- Канон-словарь уже содержит опору для якоря: `radar_v1.yaml`
  (`iff_class → iff`), `SENSOR_OBS_0001` прячет `true_transponder_id`
  как скрытую правду (спуф возможен by design).

## Decision

1. **Владелец идентичности целей = мозг (q_core_agent, WorldModel-матчер).**
   Layer-07 fusion живёт в мозге. Основание: решения (FSM/гварды)
   принимает мозг — решающий орган обязан владеть восприятием, на
   котором решает; согласовано с Safe Mode authority = Q-Core
   (`CONTEXT_LOCK` п.5, единый орган).
2. **Мост faststream_bridge = транспорт.** Не создаёт собственную
   идентичность: не матчит, не схлопывает кадр в один трек. Публичные
   `RADAR_TRACKS` несут мозговые track_id (механика доставки —
   в K2-коде; subject-контракт `radar_v1` сохраняется, no-v2).
3. **Транспондер-якорь.** Публичная идентичность якорится на
   транспондер/IFF там, где он есть. Транспондер = якорь ЗАЯВЛЕННОЙ
   идентичности, не истины: `true_transponder_id` скрыт границей
   SENSOR_OBS_0001 (hidden_truth) — отсюда наша интерпретация «спуф
   возможен by design»; расхождение заявленного и наблюдаемого —
   материал Layer-08 sensor trust, отдельное будущее решение.
4. **Переходный период (до K2-кода): двухключевой контракт intent
   (S1-мини).** Каждый intent по цели несёт пару
   `qcore_track_id + public_track_id`; двухключевой carry intents (N8)
   объявляется каноном перехода, а не локальным хаком (S1-мини должен
   починить его мёртвую public-ветку: резолв по обоим доменам, не
   только по снапшоту мозга №2). Это ГЕЙТ этапа 9b: 9b не стартует
   без S1-мини.
5. **N7 — долг:** FSM читает цели только из WorldModel; сырой
   proto-путь `_get_best_station_track` подлежит сносу в K2-код.

## Consequences

Positive:
- Одна физическая цель = одна идентичность на всех уровнях (после
  K2-кода); этап 9b и боевые этапы получают твёрдую опору.
- S3 (три шкалы смерти трека) сводится к владельцу вместе с fusion.

Negative / costs:
- K2-код — 44 файла / 4 контура (матрица карты), выполняется ПОСЛЕ
  пакета playable; до него живёт двухключевой налог.
- Мост теряет «умность» — вся ассоциация концентрируется в мозге
  (осознанная централизация).

## Alternatives considered

1) **Консоль слушает мозговые треки напрямую** — жизнеспособно, но
   рушит транспортную развязку и NATS-контракт `radar_v1` первым же
   шагом; допустимо как вариант РЕАЛИЗАЦИИ внутри K2, не как отдельное
   решение сейчас.
2) **Мост-владелец идентичности** — отвергнут: решающий орган (FSM)
   остался бы с чужой идентичностью; мост не имеет доступа к полному
   контексту мира.
3) **Отдельный fusion-сервис** — отвергнут для phase1: третий мозговой
   процесс усугубляет S2 вместо лечения.

## Пересечения (сверка «задним числом»)

- `SENSOR_OBS_0001` не ломается: граница Layer-06 (objective → sensor)
  не трогается; fusion работает ПОВЕРХ `SensorReadingSnapshot`.
- K1 snapshot-контракт консоли — независимый трек (консольные
  дериверы age/trust); порядок по карте: K5 → K1 → S1-мини → 9b.
- Пин-тесты P2 (`radar_track_visible`) остаются зелёными: смена
  владельца не меняет subject/payload-контракт до K2-кода.
- N6 (гвард дважды) закрывается ADR «владелец тика» — здесь только
  идентичность.
- Пятый путь (вне скоупа, для полноты): q-sim публикует SR-кадры
  напрямую (`q_sim_service/radar_publisher.py`, subject
  `qiki.radar.v1.tracks.sr`) — их слушает только legacy-консоль;
  ORION V не использует. Судьба пути решается в K2-коде.
