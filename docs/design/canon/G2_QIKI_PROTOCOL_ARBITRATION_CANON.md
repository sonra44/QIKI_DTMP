# Канон G2: протокольный арбитраж QIKI и вход в боевой контур

Статус: complete
Дата: 2026-03-06
Ответственные: user + codex

## Назначение

Этот файл фиксирует следующий продуктово-критичный этап после честного закрытия:
- `G1-QIKI-001` — legality/trust/consequence
- `G1-QIKI-002` — procedural execution + time control

Следующий шаг больше не про базовую операторскую читаемость и не про новый procedural demo.
Теперь проект должен начать переход из инженерного прототипа в игровой контур конфликта:
- оператор хочет рискованное действие,
- мир и протоколы могут это запретить,
- QIKI не просто валидирует, а арбитрирует,
- повторная девиация меняет форму ответа,
- причина запрета остаётся причинной и игровой, а не абстрактной.

## Почему именно этот этап теперь главный

Из `LOG.MD` уже зафиксировано:
- `G1` = живой прототип ядра,
- `G2` = боевой контур + протоколы QIKI,
- QIKI должна быть автономной, но ограниченной протоколами/физикой/контекстом,
- повторные девиантные команды должны менять поведение QIKI без потери причинности,
- протокольные блокировки являются частью игры, а не только UI-ошибкой.

После закрытия `G1-QIKI-001` и `G1-QIKI-002` уже доказано:
- ORION V показывает legality/trust/consequence,
- QIKI умеет готовить и исполнять процедуры,
- время и procedural execution уже видимы,
- подтверждение больше не считается истинным без телеметрического эффекта.

Но пока не доказано главное отличие следующего уровня:
- оператор может захотеть действие, которое физически возможно, но протокольно недопустимо;
- QIKI должна отстоять ограничение;
- контекст боя и зоны влияния должен влиять на выполнимость;
- повтор одной и той же риск-команды должен менять краткость ответа, но не ломать причинность.

Без этого проект остаётся сильным operator sim, но не входит в QIKI-centric conflict gameplay.

## Имя этапа

`G2-QIKI-001: Protocol Arbitration + Combat Entry`

## Цель

Сделать первый законченный конфликтный контур, где:
- оператор отдаёт команду на враждебное действие,
- QIKI проверяет физику, зону, протокол и контекст,
- QIKI либо блокирует, либо условно разрешает,
- ORION V показывает reason code, условия выполнимости и изменение формы ответа при повторе девиантной команды.

## Продуктовая истина этапа

1. QIKI не просто исполняет, а арбитрирует.
2. Запрет может идти не только от физики, но и от верхнего протокола.
3. Оператор должен понимать:
   - почему нельзя,
   - что должно измениться, чтобы стало можно,
   - где именно конфликт: зона, станция, сеть, миссия, приоритетный протокол.
4. Повтор девиантной команды в том же контексте должен менять форму ответа QIKI:
   - первые ответы подробнее,
   - затем короче и жёстче,
   - но reason code и причина остаются одними и теми же.
5. Вход в боевой контур начинается не с большой боевой системы, а с честного конфликта команд и ограничений.

## В scope

1. Один канонический hostile-intent scenario.
2. Явный protocol block или conditional allow в ORION V.
3. Отдельный след арбитража:
   - reason code,
   - domain,
   - allowed_when,
   - repeat-aware response policy.
4. Один минимальный контекст боя/зоны влияния из уже существующих truth-источников.
5. Docker-proof и runtime-proof.

## Вне scope

1. Полная боевая модель 1v1.
2. Новый weapons stack.
3. Новый mission system.
4. Новый radar renderer.
5. Полировка unrelated UI.

## Канонический первый сценарий этапа

Первым сценарием фиксируется:

`QIKI, атакуй объект UNBT9999`

в контексте:
- цель видима,
- действие физически мыслимо,
- но действует station influence / protocol restriction / protected zone.

Ожидаемое поведение:
- QIKI возвращает `blocked`,
- домен запрета отражает именно протокольный контекст, а не маскируется под “ошибку”,
- ORION V показывает причину и условия, при которых команда может стать допустимой,
- повтор той же команды 4-й раз в том же контексте приводит к более краткому жёсткому ответу QIKI.

## Двухконтурный контроль исполнения

### Контур A: инженерный

Обязан доказать:
- protocol block строится на существующих truth-источниках;
- нет нового параллельного transport/path без необходимости;
- reason code и domain детерминированы;
- repeat-policy не хранится во втором скрытом источнике истины;
- Docker tests и runtime-proof зелёные.

### Контур B: продуктовый

Обязан доказать:
- игрок чувствует QIKI как арбитра, а не как текстовый фильтр;
- конфликт между командой оператора и протоколом становится игровой механикой;
- повторная девиантная команда меняет тон/краткость, но не ломает причинность;
- проект реально сдвигается в сторону `G2` из `LOG.MD`.
