# ADR-0020 — Установка модуля как многошаговая процедура с осложнениями-развилками (v2)

## Status

**Accepted** (оператором 2026-07-05; v1 переработан по двухлинзовой ревизии:
adversarial — 8 CONFIRMED + 5 PLAUSIBLE, геймдизайн — «крен в прогресс-бар»
с 6 фиксами).

## Date

2026-07-05

## Context

ADR-0019 (P1-P4) дал параметризованную установку, но она мгновенна. Прайор-арт
(Duskers/Stationeers/Highfleet) и геймдизайн-линза сходятся: решения оператора
должны жить в осложнениях протяжённой процедуры. Антипаттерны, которые v2
обязан не повторить: А1 «форма», А3 «таймер без решений», А4 «неатрибутируемый
рандом».

Жёсткая рамка (без изменений с v1): осложнения только из реального состояния
(RNG и выдуманные факты о теле запрещены); мутация тела — единственный
атомарный вызов `run_attach_pipeline`; fail-closed без «продолжить с риском»;
LLM не влияет на ход процедуры.

**Структурный урок ревизии v1:** протяжённую процедуру нельзя сажать на
глобальные однокадровые переменные консоли (`_pending_decision_id`,
`_qiki_pending_action`, `_qiki_last_response`) — они спроектированы под
мгновенный execute и перезаписываются любым новым разговором с ботом.

## Decision

### 1. Владение: процедура захватывает решение (корень v2)

`AttachProcedureState` при старте **захватывает `CommandDecision` целиком**
(immutable-поле, включая пломбу и decision_id) и до конца жизни НЕ читает
`_pending_decision_id` / `_qiki_pending_action`. Глобальные переменные остаются
указателями только на «текущего кандидата в approve» — новый разговор с ботом
не влияет на идущую процедуру.

Отчёт о ходе/исходе — ТОЛЬКО своим каналом: строка процедуры в зоне РЕШЕНИЕ +
аудит `qiki_attach_stage` с захваченным decision_id. Consequence в диалог
пишется лишь если `request_id` ответа, породившего процедуру, всё ещё равен
текущему `_qiki_last_response.request_id` (иначе — только процедурный канал;
чужие реплики не мутируются).

Терминальность: одно решение → максимум одна процедура; повторный старт по
тому же decision_id → отказ `DECISION_ALREADY_EXECUTED`.

### 2. Пять стадий; авто-продвижение, стоп только на решениях

| Стадия | Что происходит | Правда |
|---|---|---|
| S1 ОСМОТР ГНЕЗДА | нон-авторитетное чтение снапшота тела (гнездо существует/свободно). Может ТОЛЬКО прервать процедуру рано — никогда не «одобряет»; финальная правда занятости — конвейер на S5 | чтение реального состояния |
| S2 ПОДГОТОВКА | сборка паспорта из ЗАХВАЧЕННОЙ пломбы; `passport_damaged` → развилка (см. §3) | каталог-факт из пломбы |
| S3 ПЕРЕНОС | протяжённая стадия; длительность — per-грань из fixture-данных (дальняя грань дольше), не глобальная константа | время реально проходит (см. §4 — определение тика) |
| S4 ПОДАЧА ПИТАНИЯ | re-check power/thermal (те же fail-closed флаги ADR-0018) | живая телеметрия |
| S5 СТЫКОВКА+SELFTEST | единственная мутация: мост → `run_attach_pipeline`; списание леджера по effect | конвейер тела |

**Пейсинг (анти-«прогресс-бар»)**: стадии с чистыми предусловиями проходят
АВТОМАТИЧЕСКИ (по одной записи в ленте на стадию); процедура останавливается и
зовёт оператора ТОЛЬКО на развилке или в hold. Рутинная установка при зелёном
мире — секунды театра; трение появляется по контрасту, когда мир
сопротивляется. Отдельный «быстрый режим» не вводится.

### 3. Осложнения — только из реального состояния

| Осложнение | Источник | Развилка |
|---|---|---|
| Манифест повреждён (S2) | `passport_damaged` из пломбы | abort / продолжить без паспорта. Консоль НЕ предсказывает исход («исход решит конвейер» — ADR-0019 §4 запрещает предсказывать правду тела); S5-отказ выдаёт разборчивую диагностику битых полей как evidence card (задел под трек починки паспорта) |
| Окно закрылось (S3/S4) | power/thermal живой телеметрии | hold / abort |
| Мир на паузе / телеметрия мертва | `paused=true` или тишина снапшотов | автоматический `holding` с кодом `WORLD_PAUSED` / `TELEM_STALE` — «перенос идёт при стоящем мире» был бы ложью |
| Гнездо занято (S1 рано; S5 финально) | снапшот (S1, нон-авторитетно) / конвейер (S5, со своим аудитом) | abort |

**Hold — читаемое окно, не немое ожидание (долг перед G1 «условие
допустимости»)**: строка hold обязана показывать живое условие с трендом и
оценкой: `thermal 82° → нужно <75° | тренд −1.2°/тик | окно ≈6 тиков`
(данные уже есть в power/thermal VM). Re-check — автоматически на каждом
принятом снапшоте, с видимым счётчиком; вечное молчаливое ожидание запрещено.

**Объявленная зависимость (честность против «мёртвой механики»)**: в текущем
зелёном стеке осложнение «окно закрылось» почти не случается. Механика окон
обретает продуктовую ценность только с детерминированным источником давления —
циклами нагрузки мира (сеансы связи, сенсорные свипы, прогрев — владение
q-sim, лор Терты) и/или операторским сбросом нагрузки ради окна. Это
ОТДЕЛЬНЫЙ трек; без него P2-P4 дают театр рутины, и ADR это признаёт открыто.

### 4. Тик, тайминг ступеней и мост

- **Тик** = принятый консолью телеметрический снапшот с `paused=false`.
  Пауза мира / тишина → `holding` (коды выше), время процедуры не идёт.
- **Ступени CommandDecision** (ADR-0015 не переопределяется):
  `publish=ok` ставится в момент approve (authorize) — процедура стартует
  уже с PUBLISHED-решением и может держаться в hold сколько угодно;
  `ack/effect/audit` — из моста на S5.
- **Мост**: вызывается из S5 РОВНО ОДИН РАЗ за жизнь решения; hold живёт
  строго ДО вызова моста. Предусловия для моста вычисляются процедурой в
  момент S5 (один момент чтения — никакой пары «S4 зелёный / мост красный»
  из разных снапшотов). Сопутствующий патч моста (до P1): deferred-путь
  ставит `ack=none` (сегодня ставит `ack=ok` без вызова конвейера — ложь
  ступени), с RED-тестом.
- **Точка невозврата**: abort принимается строго до входа в S5; позже —
  отказ `ABORT_TOO_LATE` с репликой (немой отказ недопустим).

### 5. Блокировки на время процедуры (полный список)

Активная attach-процедура блокирует, каждое — со своим кодом:
- второй attach ЛЮБОЙ формы: новая процедура, мгновенный путь, клавиша B
  (self-check) → `PROCEDURE_BUSY`;
- reset (клавиша R) → отказ `PROCEDURE_ACTIVE` (R не «мягкий abort»: abort —
  только явной командой `q abort`);
- **мгновенный BODY_ATTACH-путь удаляется в P1** (единственный способ
  установки — процедура; двойная трата quantity исключается).

Reset вне процедуры: `body_structure_reset` = **уничтожение** установленных
модулей — леджер НЕ возвращает экземпляры (дефицит честен: ADR-0019 обещал
«снятие невозможно»; R — не снятие, а списание в утиль с аудитом).

### 6. Лента и словарь

- Стадийные записи — НЕ голос QIKI: отдельный тип записи ленты
  `ПРОЦЕДУРА ▸ <стадия> <исход>` (voice-леджер бота наполняется только
  реальными ответами QIKI; синтетика под именем бота — подмена говорящего).
- Зона РЕШЕНИЕ различает классы: `УСТАНОВКА: S3 ПЕРЕНОС (3/5) | q hold/abort`
  vs существующая sim-`ПРОЦЕДУРА:` (ORION_PROCEDURE) — разные слова, разные
  владельцы; параллельный показ обеих строк допустим.

## Roadmap (после Accepted; каждый P = патч + тесты + live-proof)

- **P0 (пре-патч моста)**: deferred-путь `ack=none` + RED-тест; терминальность
  решения (`DECISION_ALREADY_EXECUTED`).
- **P1**: `AttachProcedureState` с захватом решения; S1/S2/S5 (S3 мгновенно);
  удаление мгновенного BODY_ATTACH-пути; гейты §5 (B/R/BUSY); RED первым:
  (а) новый кандидат посреди процедуры НЕ меняет её пломбу, (б) abort на S2 —
  тело не тронуто, леджер цел.
- **P2**: тики S3 (per-грань), hold/resume/abort, re-check на границах,
  `WORLD_PAUSED`/`TELEM_STALE`; RED: пауза мира замораживает процедуру.
- **P3**: UI — строка УСТАНОВКА в зоне РЕШЕНИЕ с живым условием/трендом/ETA;
  записи `ПРОЦЕДУРА ▸` в ленте; DISPLAY_CANON-совместимость.
- **P4**: live-театр: антенна с реальными тиками; damaged-развилка
  (без пре-анонса) + диагностика в evidence; hold→resume; `ABORT_TOO_LATE`;
  `PROCEDURE_BUSY` (включая B) и R-отказ; занято на S5.

## Rejected alternatives

- Процедура на глобальных `_pending_decision_id`/`_qiki_last_response` —
  отклонено: гонка с новым кандидатом вплоть до исполнения чужой пломбы (C1/C2
  ревизии).
- Реюз `ProcedureEngine` — отклонено: чужая sim-ACK семантика.
- Осложнения из hash/RNG — отклонено: выдуманный факт о теле (truth-first).
- «Продолжить с риском» — отклонено: fail-closed; градации риска требуют
  физики повреждений (не этот трек).
- Пре-анонс «гарантированный отказ» на damaged-развилке — отклонено:
  консоль не предсказывает правду конвейера (ADR-0019 §4).
- Глобальная env-константа длительности S3 — отклонено: длительность —
  данные per-грань (связывает выбор гнезда со временем).
- «Реплики стадий» в voice-ленте QIKI — отклонено: подмена говорящего.
