# F1 — переработка под игровое поле (дизайн-аудит)

Статус: ПРЕДЛОЖЕНИЕ (на согласовании с оператором)
Дата: 2026-07-04
Грунт: `docs/design/game/qiki_operator_lore_notes.md`, `bot_gdd.md`, `SECTOR_TERTA_CANON.md`,
живой кадр F1 (телеметрия включена, мир RUN), ADR-0014/0015/0016.

## 0. Оптика: кто смотрит и зачем

Игрок — Оператор, подключённый к автономному боту QIKI «как с Земли через терминал»
(Сектор Терта: радиация, аномалии, угрозы). F1 — взгляд за 3–5 секунд, отвечающий
на **пять вопросов оператора**:

1. **ЧТО с ботом?** — тело, ресурсы, пределы (жизненные показатели).
2. **ЧТО вокруг?** — пространство, треки, угрозы, цель (восприятие).
3. **ЧТО бот делает и почему?** — намерение QIKI, фаза полёта, спор/согласие.
4. **ЧТО решать мне сейчас?** — гейты, подтверждения, риск ближайшего действия.
5. **ЧТО я потеряю, если умру сейчас?** — хардкор-мета: несинхронизированные данные мира.

Каждая строка F1 обязана обслуживать один из этих вопросов. Строка, отвечающая только
на вопрос разработчика («работает ли рендер?») — не для F1.

Правила, которые НЕ трогаем: честность §19 (source/freshness/trust), «коды остаются
кодами», single-owner rule, канон-стрипы ADR-0016.

## 1. Разбор по зонам (сверху вниз)

### Z1. MISSION CONTROL STRIP — `МИР | СВЯЗЬ | АКТУАЛ | СЕНС | УПР`
- **Игровой смысл:** жив ли контур связи с ботом = сама возможность играть. Работает.
- **Нюанс:** `МИР RUN/PAUSE/STOP` — **мета-уровень симуляции** (пауза игры), не диегетика
  оператора. Оставить (нужна пауза), но в tooltip проговорить: «состояние симуляции».
- **Гэп (диегетика связи):** EMCON-состояние и канал (radio/NBL) — GDD §14. → G-E/F7.
- Вердикт: **ОСТАВИТЬ**, tooltip-уточнение.

### Z2. SAFETY & HEALTH STRIP — `ALRT · SAFE` + чипы `PWR/THRM/PROP/HULL/CPU/QIKI`
- **Игровой смысл:** жизненные показатели. Живые значения уже играют
  (`PWR OK Заряд 93%, 27.6В, ~245 мин` — таймер жизни!).
- **Гэп (ключевой ресурс GDD):** **SoC_cap** (конденсаторы) с гейтами
  `T_boost=0.6 / T_hold=0.3` — это РАЗРЕШЕНИЕ манёвров (буст можно/нельзя), главный
  тактический ресурс. Сейчас囚 в POWER-строке правого MFD. → в чип PWR добавить
  cap-аспект: `PWR | OK | bat 93% · cap 70% ▸буст` (код гейта: `БУСТ/ДЕРЖ/СТАБ`).
- Вердикт: **ДОРАБОТАТЬ** (cap-гейт в чип PWR).

### Z3. Строка MFD-статуса — `РЕЖИМ: КОКПИТ | КОРПУС: каркас (посев) | модулей: 0 | УЛИКИ: waiting | ИСТОЧНИК: аудит/локальный посев`
- **Проблема:** наполовину внутренняя кухня evidence-станции. «УЛИКИ: waiting |
  ИСТОЧНИК: аудит/локальный посев» — жаргон пайплайна; доверие экрану уже показано
  СЕНС/АКТУАЛ (single-owner!). «КОРПУС/модулей» дублирует правый MFD.
- **Игровая замена — строка ИДЕНТИЧНОСТИ бота** (вопрос «кто я»):
  `QIKI-<серийник> | додекаэдр · 12 граней | модулей 0/12 | реплика #N` (реплика — лор
  смерти/репликации; серийник из hardware_profile_hash пока нет id).
- Вердикт: **ПЕРЕРАБОТАТЬ** (identity-строка; evidence-мета → tooltip).

### Z4. Левый MFD — кнопки `РАДАР/НАВ/ЦЕЛЬ/СЕКТОР/МИССИЯ`
- **Проблема:** страница РАДАР при пустом эфире показывает выжимку статусов
  («Общий статус/Контекст миссии…») — чужой контент. Радар обязан быть радаром.
- **Игровая переработка (R.L.S.M, GDD §12):**
  - треки: `пеленг | дальность | скорость | IFF | качество | риск` (модель
    RadarTrackModel уже несёт всё это, показываем 5 полей из ~20);
  - пусто → `эфир чист | охват 360° | режим: НАВИГАЦИЯ` (не чужие статусы);
  - режим восприятия (`НАВИГАЦИЯ/СБЛИЖЕНИЕ/КАРТОГРАФИЯ/НАУКА/SAFE`) — target-only,
    честно метить пока q-sim не отдаёт.
- Кнопочный набор внешнего мира — осмысленный. Вердикт: **ПЕРЕРАБОТАТЬ страницу РАДАР**.

### Z5. Правый MFD — кнопки `СИСТ/СЕНС/ПИТ/ТЕПЛО/СВЯЗЬ/ДВИГ/СТЫК/ЖУРН/ПРОЦ`
- Тело бота, «Физические последствия» (масса/ЦМ/инерция) — ровно GDD-механика модулей.
- Вердикт: **ОСТАВИТЬ** (мелочь: subtitle-рамки перевести хвосты, см. Z6).

### Z6. SOFTKEYS / ПЕРЕХОДЫ (4 ряда)
- Ряды 1–2 (`Маршрут OK · F2 …`) — glance+goto, работает. **ОСТАВИТЬ**.
- Ряд 3 (`◀ Действие | Предпросмотр · ПРОВЕРКА КОРПУСА | Применить | ▶`) — колесо
  обучающего цикла Ф1; действия фиксированы (5 шт.) и не зависят от сцены.
  **ПЕРЕРАБОТАТЬ → контекстные действия сцены** (см. Z7/G-D): docked → «Расстыковка /
  Зарядка / Диагностика»; free_flight → «Burn / Brake / Удержание»; при цели →
  «Сближение / Наблюдение». Это ядро геймплея.
- Ряд 4 (`Panel ▲ / Help · ON / Panel ▼`) — ОСТАВИТЬ.
- Подпись рамки `left/right MFD restored` — англ. хвост → «перекл. страниц: [ ]».

### Z7. Панель QIKI / ОПЕРАТОР (включая «F1 / Первый живой цикл»)
Самая перегруженная зона; сейчас — обучалка+приёмка, а должна быть **сердцем игры**.
- **УБРАТЬ с экрана (в тесты/Help):**
  - `ПАНЕЛИ | готово 6/6` + `КОРПУС ✓ | ПИТАНИЕ ✓ …` — acceptance-чеклист разработчика;
  - `СОБЫТИЯ: готово | история: 0` (дубль ЛЕНТЫ);
  - строка `цикл: снимок → экран → …` — раз в Help, не каждый кадр;
  - `КЛАВИШИ | ←/→ …` — уже есть Help·ON и подсказки; держать только при Help.
- **СВОРАЧИВАТЬ по H (Help·OFF по умолчанию после первого применённого действия):**
  СПРАВКА/ПАЛИТРА/ПОДСКАЗКА-строки.
- **ГЛАВНЫЙ ГЭП — ГОЛОС QIKI.** Панель называется QIKI, но QIKI молчит. По лору QIKI —
  компаньон: докладывает, спорит с плохими командами (потом выполняет), не понимает,
  что такое «человек». Реплики уже приходят (`qiki_response.reply.ru`) → диалоговая
  лента `QIKI ▸ …` (последние 2–3 реплики) вместо статус-рядов LEGALITY/TRUST в
  сыром виде (коды — в tooltip/F6). Тон — машинный, по SECTOR_TERTA §5.
- `— граница: … | источник | доверие` — оставить одну строку (честность), сжать.
- Вердикт: **КРУПНАЯ ПЕРЕРАБОТКА** (декомпрессия обучалки + голос QIKI).

### Z8. Тело кокпита (СИСТЕМА/BODY/ФИЗИКА/POWER + 5 панелей + «Краткие факты»)
- Каркас панелей (Общий статус/Контекст миссии/Наведение/Инциденты/Маршрут) — рабочий.
- «Краткие факты» — dim-простыня с повторами `Нет данных` → **показывать только
  непустые строки**, схлопывать пустые группы одной строкой `…нет данных: связь, орбита`.
- `BODY STRUCTURE | online | faces=12 …` — англ. код-строка в русском поле: метки
  оставить кодами, но заголовок строки → `КОРПУС(посев)` для консистентности с ФИЗИКА/POWER.
- Вердикт: **ПОЛИРОВКА**.

### Z9. ACTION RAIL
- Терсе-коды `M LIVE | CMD | P | ACT` — коды остаются. КОНСОЛЬ-лента — журнал, играет.
- Кнопки `Инц ← / Инц → / Подтв. / Снять` при 0 инцидентов — мёртвые → **скрывать
  при пустоте** (canon show-when, как УПР).
- Вердикт: **МЕЛКАЯ ДОРАБОТКА**.

## 2. Чего НЕТ для игры (гэпы по GDD/лору, по весу)

| # | Гэп | Канон | Что нужно |
|---|-----|-------|-----------|
| G1 | **Синхронизация данных со станцией** — «что я потеряю при смерти»: последний sync, объём несинхронизированного, зона приёма | лор §4 (хардкор-ядро) | дизайн+runtime, свой ADR; на F1 — одна строка `SYNC` в стрипе/статусе |
| G2 | **Фаза полёта Burn/Coast/Brake** + вектор/скорость | GDD §5 | q-sim частично умеет (rcs.fire); строка полёта в «Наведение» |
| G3 | **SoC_cap гейт** `БУСТ/ДЕРЖ/СТАБ` (T_boost/T_hold) | GDD §4 | данные есть (supercap_soc_pct) — только показ+пороги |
| G4 | **BRAKE OVERRIDE** — «большой красный», одна клавиша | GDD §8 (наивысший приоритет) | команда+кнопка (через _publish_sim_command; q-sim: sim.rcs.* / TBD) |
| G5 | **Риск сближения (TCAS)** — время до сближения, класс угрозы | GDD §12 | треки уже несут range/vr → derived risk |
| G6 | **Голос QIKI** (диалог, спор, доклады) | лор §2–3 | reply уже приходит → лента (Z7) |
| G7 | **EMCON/сигнатура** — излучение=заметность | GDD §14 | телеметрия имеет EMCON_state (сейчас missing) — честный код в СВЯЗЬ-tooltip/F7 |
| G8 | **Режим восприятия R.L.S.M** | GDD §12 | target-only, честно метить |

Все гэпы вводим ЧЕСТНО: пока нет источника — target-only пометка или не показываем;
никакого fake-руна.

## 3. Фазы переработки (каждая: дизайн→правка→тест→живой кадр→коммит)

| Фаза | Содержание | Размер |
|------|-----------|--------|
| **G-A. Декомпрессия обучалки** | Z7-уборка (ПАНЕЛИ 6/6, дубли, клавиши→Help), Z8 «Краткие факты» без пустых, Z9 скрытие мёртвых кнопок, Z6 подпись рамки | S |
| **G-B. Радар — это радар** | Z4: страница РАДАР = треки (пеленг/дальность/скорость/IFF/риск G5) + «эфир чист», без чужого контента | M |
| **G-C. Голос QIKI** | Z7: диалоговая лента QIKI ▸ реплики; коды LEGALITY/TRUST → tooltip/F6 | M |
| **G-D. Контекстные действия + полётный контур** | Z6-ряд3 → действия сцены; G2 фаза полёта; G3 cap-гейт (Z2 чип); G4 Brake override | L |
| **G-E. Хардкор-мета SYNC** | G1: дизайн синхронизации данных (ADR) + строка SYNC | L (ADR) |
| **G-F. Идентичность бота** | Z3: identity-строка QIKI-id/реплика #N | S |

Рекомендуемый порядок: **G-A → G-B → G-C → G-F → G-D → G-E**
(сначала расчистить поле, потом наполнить миром, голосом и смыслом).

SECTOR_TERTA решаются в пользу канона. Каждая фаза перед реализацией сверяется с
