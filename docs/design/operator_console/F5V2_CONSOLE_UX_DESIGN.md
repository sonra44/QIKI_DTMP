# F5 v2 + Консоль UX — свободный чат, дисплей QIKI, тихий журнал, кнопки вместо писанины (v2)

Статус: **Accepted** (2026-07-06, приоритеты делегированы оператором; v1
переработан по трёхлинзовой ревизии Opus/xhigh: adversarial — 8 CONFIRMED,
геймдизайн — «движок принять, кожу ре-диегетизировать», аудит F7 — вердиктом).
Порядок исполнения: LLM-ветка диалога (пре-W7, без стриминга) → W1+W2 →
F7-багфиксы попутно → W3+.
База: исследования Textual 8.2.7 + UX-практик (Elia/oterm/Claude Code/Codex;
Duskers/Mudlet/Cogmind/DCSS; ISA-101/18.2 HMI).
Подчинён: DISPLAY_CANON (T0-T9, языковая рамка, 8в), F5-док §1, ADR-0015..0020.

## 0. Требования оператора (дословно)

1. Шум действий — в отдельный сворачиваемый журнал с таймстампами.
2. Масштабирование зон и переключение фокуса.
3. F5 — свободный чат: подсветка, CSS, явное форматирование реплик.
4. F5 пополам: справа дисплей, куда QIKI выводит инфу (сначала каркас).
5. «Писанина кроме общения — смерть»: команды видны и жмутся.
6. Ввод/вывод консоли — пересмотреть как дизайн.
7. Игровая активная пауза — сделана отдельно (кнопка ⏸/▶ Мир, реестр 9а).

## 1. Пять решений-инвариантов v2 (из ревизии; нарушать нельзя)

**И1. ОДИН владелец ввода на F5** (закрывает C1/C2): постоянное чат-поле [1]
И ЕСТЬ командный вход экрана. Роутинг по грамматике ввода, детерминированный,
в консоли (не LLM):
- `q confirm | q abort | q resume | q hold | q cancel` (bare-глаголы) → control;
- `/…` → команды консоли (`пауза`, `старт`, `help`, `f1..f8`);
- всё остальное → intent боту (без `q:`-префикса).
Глобальный командный режим рейла на F5 отключается. Все строки-подсказки
(«шаг: q confirm», «▶ q resume») остаются ПРАВДОЙ — эти глаголы работают из
чат-поля.

**И2. Клавиши против чата** (закрывает C3): пока фокус в чат-поле, printable-
клавиши — это ТЕКСТ. Хоткеи зон/чипов работают только вне фокуса ввода;
прыжки фокуса — `Alt+1..4` (модификаторный слой, Input не съедает); Esc —
явная state-машина: (1) стрим ответа идёт → прервать стрим; (2) фокус в
вводе → фокус в командный слой; (3) иначе → закрыть модалку/ничего.
Чипы [3] — mouse-first, клавиши — дубль вне ввода.

**И3. Глобальный фидбек живёт** (закрывает C4/T5): стрип КОНСОЛЬ/CONSOLE
НЕ сносится вслепую. Разводим двух владельцев: (а) **статус-строка** рейла
(1 строка: last_command/help-фидбек) — глобальная, остаётся; (б) **журнал
событий [4]** — глобальный виджет (один на все экраны, ответ на вопрос v1
§5.2), на F5 встроен зоной, на прочих экранах — вызывается клавишей.
Фильтрация — ТОЛЬКО на отображении; путь в аудит (EVENTS_AUDIT/F6/F8) не
трогается вообще (evidence не теряется, T4); классификация уровней — в
существующей event/severity VM (один владелец), не в UI.

**И4. Markdown — только свободному тексту; коды и штампы — plain**
(закрывает C6 + коллизию с 8в): единый голос QIKI на F1 и F5 — грамматика
штампа `QIKI ▸ <ts> <ACK|REJECT|INFO> | <текст>`; коды `[…]` и строки
ПРОЦЕДУРЫ рендерятся plain Text (урок a58fd97 не откатывается). Markdown-
виджет получает ТОЛЬКО тело свободного ответа (LLM-путь, V-этап стриминга);
подсветка ЯЗЫКОВ ПРОГРАММИРОВАНИЯ отключена — моноширинные кодовые блоки
как «машинные факты» (dim), без syntax-rainbow (диегезис: станция, не IDE).
Порядок лент — осознанное расхождение форматов при едином штампе: F1-компакт
8в — новейшее СВЕРХУ (3 строки), F5-чат — хронология ВНИЗ (полный текст);
фиксируется в DISPLAY_CANON при реализации.
Статус бота — машинный регистр: `⠋ обработка… 3s · Esc — прервать`
(станция не «думает» и не эмотирует).

**И5. Дисплей [2] — один владелец, никакого provider-markdown**
(закрывает C7/CaMeL): дисплей рисуется ТОЛЬКО из локальных view-model/
каталога по детерминированному словарю intents («покажи отсек/корпус/тренд
THRM/обстановку/действие»); свободный markdown от провайдера на дисплей НЕ
попадает никогда. Каждый элемент несёт source/freshness как любая зона.
Классификатор «покажи X» — детерминированный словарь консоли/policy, не LLM.

## 2. Архитектура экрана F5 v2 (уточнённая)

Зоны как в v1 ([1] ЧАТ | [2] ДИСПЛЕЙ / [3] КОМАНДЫ / [4] ЖУРНАЛ), с
поправками И1-И5. Дополнительно:

- **[1] ЧАТ**: асимметрия голосов (оператор — компактная строка `я ▸`;
  QIKI — штамп 8в + тело; свободное тело — Markdown без syntax-rainbow);
  anchor()-автоскролл; технические коды реплики — свёрнутая plain-строка
  `▸ коды`; **история ввода (стрелка вверх) и автодополнение id** — печатная
  строка остаётся фактурой оператора (Duskers), чипы её не заменяют.
- **[2] ДИСПЛЕЙ**: **первый payload — Trust/Legality-карта текущего
  действия** (источник/state/confidence/reason_code/условие разблокировки —
  спина G1, уже посчитана policy); затем обстановка (радар-сводка), отсек,
  тренд THRM/PWR (`textual-plot` — ПОСЛЕ проверки braille в tmux/SSH),
  карта активной процедуры установки. HMI-серость: норма серая, цвет —
  отклонениям.
- **[3] КОМАНДЫ**: инвариант — **чип = акселератор, не единственная дверь**;
  поле ввода не прячется никогда; **консеквентные команды шины
  (confirm/undock) в 1 клик НЕ сворачиваются** — только церемония
  ввода/диалога подтверждения (T4, G1). Владелец паузы мира — ОДИН: кнопка
  рейла ⏸/▶ Мир (реестр 9а); чип [П] в F5 не дублируется (канон №7).
- **[4] ЖУРНАЛ**: компакт-критерий геймплейный — «оператор сослался бы на
  эту строку потом?»: последствия/переходы состояний, новые угрозы и
  деградация trust, МОМЕНТ РАЗБЛОКИРОВКИ действия, вехи процедур,
  WORLD_PAUSED (T9-метка). ALERT авто-всплывает даже в свёрнутом. Рутина —
  счётчики (`ACK ×214`).

## 3. Этапность (сцеплена с M-гейтами; реордер по геймдизайн-линзе)

| Шаг | Что | Гейт/предусловие |
|---|---|---|
| **W1** | Каркас F5 v2: сплит [1]|[2], чат-рендер (штамп+асимметрия+anchor), ввод по И1/И2 | нулевые внешние зависимости; write-путей нет (M-гейты не задеты) |
| **W2** | **Trust/Legality-карта на дисплее** — первый видимый игроку выигрыш | локальные VM |
| **W3** | Журнал [4] глобальный + статус-строка рейла (И3); фильтр на отображении | аудит не трогается |
| **W4** | Чипы [3] per-screen + Footer + `?`-шпаргалка + автодополнение/история ввода | И-инварианты чипов; textual-autocomplete после tmux-проверки |
| **W5** | Дисплей-словарь: отсек/обстановка/процедура; тренды `textual-plot` | plot после tmux/SSH-проверки braille |
| **W6** | Фокус Alt+цифры, подсветка, ресайз биндингами; drag-сепаратор (опц.) | — |
| **W7** | Стриминг LLM (MarkdownStream) | ПОСЛЕ ключа + audit-недели gateway (M-серия) |

Каждый W = патч + тесты + live-proof (herdr-PTY канон).

| `textual-plot` | кандидат W5 | обязательная проверка braille в tmux/SSH ДО включения |

## 5. Сопутствующий вердикт: F7 (аудит 2026-07-06)

F7 сегодня — плоский debug-дамп: 11 равновесных строк, замороженные `0.0 мс`
(ВРЕДНАЯ: читается «мгновенно», означает «не мерилось»), дубли (queue↔store,
NATS↔стрип, replay↔хедер), сырые литералы `False`/`connected`, съеденный
markup'ом якорь `[F7]`. Решение (фиксируется строкой реестра DISPLAY_CANON,
реализуется отдельным малым слайсом): идентичность F7 = **СОСТОЯНИЕ
ТЕРМИНАЛА** (контур консоли: КАНАЛ/ПОТОК/ОТКЛИК/РЕСУРС — не дубль F2-тела);
dark cockpit — в норме одна строка `КОНТУР КОНСОЛИ: OK`, разворачивается при
WARN/CRIT; коды+пороги+цвет вместо голых чисел; NODATA вместо замороженных
нулей; дедуп владельцев. Быстрый попутный багфикс (можно сразу): рендер тела
через `Text(...)` (вернуть видимый якорь) + `tr()` для «Загрузка процессора».
