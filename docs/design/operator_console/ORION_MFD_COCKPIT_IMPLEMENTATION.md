# ORION V MFD Cockpit implementation slice

Этот документ фиксирует первый безопасный implementation-slice для F1 ORION V cockpit.

## Что реализуется

- Верхняя строка статуса: телеметрия, симуляция, SAFE MODE, энергия, тепло, сенсоры, QIKI, инциденты.
- Левый MFD: РАДАР / НАВ / ЦЕЛЬ / СЕКТОР / МИССИЯ.
- Правый MFD: СИСТ / СЕНС / ПИТ / ТЕПЛО / СВЯЗЬ / ДВИГ / СТЫК / ЖУРН / ПРОЦ.
- Нижний канал QIKI: решение, причина, trust signals, consequence, telemetry confirmation, ручное подтверждение.
- Sensor inventory без магии: отсутствующие источники показываются как `Нет данных`, включая spectrometer.
- OperatorUsefulFact: выводит не просто значения, а смысл, влияние на действия, следующий шаг и источник.

## Что намеренно не тронуто

- `q_sim_service`
- физика мира
- генерация сенсоров
- NATS subjects
- QIKI decision logic
- actuation/execution

## Инварианты

- Интерфейс на русском.
- Нет моков и фейковых сенсоров.
- Каждое видимое значение имеет источник или явно показывает отсутствие источника.
- Главный экран показывает не таблицу ради таблицы, а операторскую пригодность состояния.
- Если данных нет, выводится `Нет данных`, а не придуманное значение.

## Проверка

Implementation-slice должен проверяться через:

```bash
python -m pytest tests/unit/test_orion_v_cockpit.py
```
