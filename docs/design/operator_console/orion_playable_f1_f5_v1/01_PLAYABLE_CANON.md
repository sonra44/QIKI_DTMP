# 01. Определение «playable v1»

Статус: target spec (подчинён G1-канону).

## Основание

- Игровой цикл (канон G1): `Наблюдение → Запрос/Команда QIKI → Legality/Trust → Consequence`
  с тремя обязательными UX-поверхностями: Action Legality, Sensor Trust,
  Consequence Confirmation (`docs/design/canon/G1_QIKI_OPERATOR_LOOP_CANON.md`).
- Пять вопросов оператора для F1 (`F1_GAME_FIELD_REWORK.md §0`):
  1. ЧТО с ботом? 2. ЧТО вокруг? 3. ЧТО бот делает и почему?
  4. ЧТО решать мне сейчас? 5. ЧТО я потеряю, если умру сейчас?
- Truth/Quality contract (LOG.MD, Глобальный План v2): каждое поле несёт
  `value / state / quality / confidence / reason_code / source / updated_at`;
  `N/A` — только ошибка реализации/контракта, никогда не штатный игровой статус.

## Критерии «playable v1» (все проверяемые)

| # | Критерий | Проверка |
|---|----------|----------|
| P1 | При живом стеке (nats + q_sim_service + q_core_agent + консоль) оператор за ≤5 секунд с экрана F1 получает ответы на вопросы 1–4 (вопрос 5 / SYNC — вне v1) | приёмка `07_ACCEPTANCE_CRITERIA.md` §F1 |
| P2 | Радар не лжёт: живой контакт на шине никогда не даёт «0 целей» в консоли и в мозге | Блок 0.1/0.2/0.9 + smoke `radar_track_visible` |
| P3 | Любой запрос/команда оператора имеет наблюдаемые legality/trust/consequence; исполнение достигает тела только через пломбу `CommandDecision` (F5-контур) | `06_COMMAND_SURFACE_CONTROL_PATH.md`, m5-спуф-тесты |
| P4 | Существует ≥1 команда с физическим последствием, видимым в телеметрии: release dock (уже есть, `scripts/prove_orion_v_qiki_release_dock.sh`); движение RCS — этап 10 | приёмка §F1-контекст, §RCS |
| P5 | 30+ минут runtime-smoke без падений консоли, утечек памяти и рассинхрона (DoD G1, LOG.MD:483) | smoke `runtime_30min` (этап 11) |

## Явно вне «playable v1»

- Бой (G2-гейт), миссии/нелинейность (G3-гейт).
- SYNC-мета «что я потеряю при смерти» (нужен отдельный ADR, гэп G-E).
- EMCON/сигнатура (G7), режим восприятия R.L.S.M как runtime (G8 — только
  target-only метка).
- BRAKE OVERRIDE как прямой путь мимо диалога (открытый вопрос Q8 в
  `_support/CLARIFICATION_REQUEST_001.md`).
- Security-контуры Д1/Д2 (v2 auth_context, наружный NATS-порт) — отдельный трек.

## Правила честности (наследуются, не пересматриваются)

- Здоровый сенсор — точные данные; деградировавший — неопределённость;
  отказавший/выключенный — причина отсутствия данных (README, §Product Frame).
- Пороги живут только у единого владельца `qiki/shared` (Срез 0 SelfModel);
  новые локальные копии порогов запрещены.
- Коды остаются кодами; честность §19 (source/freshness/trust) не трогаем
  (`F1_GAME_FIELD_REWORK.md §0`).
