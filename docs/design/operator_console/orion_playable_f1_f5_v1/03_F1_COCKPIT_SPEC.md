# 03. F1 «Кокпит» — спецификация переработки

Статус: target spec. Основание: `F1_GAME_FIELD_REWORK.md` (предложение
2026-07-04) — здесь его зоны переведены в статус «принято к реализации»
с конкретикой файлов. Структура зон Z1–Z9 — из исходного предложения.

Правила, которые НЕ трогаем: честность §19 (source/freshness/trust),
«коды остаются кодами», single-owner rule, канон-стрипы ADR-0016.

## Z1. Mission Control Strip — `МИР | СВЯЗЬ | АКТУАЛ | СЕНС | УПР`

- Вердикт: **оставить**. Дополнение: tooltip у `МИР` — «состояние симуляции»
  (мета-уровень, не диегетика).
- Файлы: `orion_v/widgets/` (стрип), правка только tooltip-текста.

## Z2. Safety & Health Strip — чипы `PWR/THRM/PROP/HULL/CPU/QIKI`

- Вердикт: **доработать**: cap-аспект в чип PWR:
  `PWR | OK | bat 93% · cap 70% ▸БУСТ` (гейт-коды `БУСТ/ДЕРЖ/СТАБ`).
- Данные уже есть: `supercap_soc_pct`. Пороги `T_boost=0.6 / T_hold=0.3` —
  **только из владельца `qiki/shared`**; новая локальная копия запрещена
  (дефект 0.17 как анти-образец). Если порогов в shared нет — добавить их
  именно туда, одним PR с потребителем.
- Гэп G3 из F1_GAME_FIELD_REWORK закрывается этим пунктом.

## Z3. Строка MFD-статуса → строка идентичности

- Текущее: `РЕЖИМ: КОКПИТ | КОРПУС: каркас (посев) | модулей: 0 | УЛИКИ: waiting | ИСТОЧНИК: аудит/локальный посев` — внутренняя кухня evidence-пайплайна.
- Целевое: `QIKI-<серийник> | додекаэдр · 12 граней | модулей N/12`
  (серийник из `hardware_profile_hash`, пока нет отдельного id; «реплика #N»
  — target-only до появления источника). Evidence-мета → tooltip.
- Гэп G-F.

## Z4. Левый MFD, страница РАДАР

- Текущее: при пустом эфире страница показывает чужую выжимку статусов.
- Целевое:
  - треки: `пеленг | дальность | скорость | IFF | качество | риск`
    (RadarTrackModel уже несёт эти поля);
  - риск сближения — derived из range/vr (гэп G5), помечается `source=derived`;
  - пусто → `эфир чист | охват 360° | режим: НАВИГАЦИЯ`;
  - режим восприятия R.L.S.M — **target-only** метка, пока q-sim не отдаёт.
- Файлы: `orion_v/mfd_page_content.py`, `orion_v/mfd_layout.py`,
  view-model радарной страницы (новый файл по образцу
  `cockpit_playable_view_model.py`).
- Зависимость: дефекты 0.1/0.2/0.9 (иначе страница честно покажет ложь).

## Z5. Правый MFD

- Вердикт: **без изменений** (мелочь: англ. хвосты subtitle-рамок → русские,
  вместе с Z6-подписью).

## Z6. Softkeys, ряд 3 → контекстные действия сцены

- Текущее: 5 фиксированных учебных действий «снимок → экран → … → улика»,
  не зависят от сцены; apply пишет только audit-событие
  (`app.py:880`, `f1-loop:*`).
- Целевое: действия зависят от сцены:
  - `docked` → «Расстыковка / Диагностика»;
  - `free_flight` → «Burn / Brake / Удержание» (доступно после 0.10);
  - выбранный трек → «Сближение / Наблюдение».
- Механика исполнения — строго по `06_COMMAND_SURFACE_CONTROL_PATH.md`:
  выбор действия публикует **intent** в `qiki.intents`; кандидат,
  предпросмотр и подтверждение остаются в F5-контуре.
- Учебный цикл (5 действий) переезжает в Help-режим/обучалку, перестаёт
  занимать ряд 3. Пин-тест меняется синхронно (риск R1).
- Подпись рамки `left/right MFD restored` → `перекл. страниц: [ ]`.
- Файлы: `cockpit_playable_view_model.py` (модель действий),
  `app.py` (`action_cockpit_playable_*`, `app.py:803-952`), softkey-виджет.

## Z7. Панель QIKI / ОПЕРАТОР — декомпрессия + голос

- **Убрать с экрана (в тесты/Help):** `ПАНЕЛИ | готово 6/6`,
  `КОРПУС ✓ | ПИТАНИЕ ✓ …` (acceptance-чеклист разработчика),
  `СОБЫТИЯ: готово | история: 0` (дубль ленты), строку
  `цикл: снимок → …`, блок `КЛАВИШИ` (держать только при Help·ON).
- **Голос QIKI (гэп G6):** лента `QIKI ▸ …` — последние 2–3 реплики из
  `qiki_response.reply.ru` (источник живой: `orion_v/qiki_voice.py`);
  сырые статус-ряды LEGALITY/TRUST → tooltip/F6, на панели — одна строка
  честности `— граница: … | источник | доверие`.
- Тон реплик — машинный, по SECTOR_TERTA §5 (не трогаем генерацию,
  только показ).
- Файлы: `screens/cockpit.py` (панель), `qiki_voice.py`,
  `cockpit_playable_view_model.py`.

## Z8. Тело кокпита — «Краткие факты»

- Показывать только непустые строки; пустые группы схлопывать одной строкой
  `…нет данных: связь, орбита`. Заголовок `BODY STRUCTURE …` → `КОРПУС(посев)`
  (метки-коды остаются кодами).

## Z9. Action Rail

- Кнопки `Инц ← / Инц → / Подтв. / Снять` при 0 инцидентов — скрывать
  (canon show-when, как УПР). Терсе-коды `M LIVE | CMD | P | ACT` остаются.

## Явно вне v1 (для F1)

- G-E SYNC («что я потеряю при смерти») — требует отдельного ADR.
- G4 BRAKE OVERRIDE — см. Q8 в `_support/CLARIFICATION_REQUEST_001.md`.
- G7 EMCON — честный код в tooltip СВЯЗЬ/F7, без выдуманного источника.
