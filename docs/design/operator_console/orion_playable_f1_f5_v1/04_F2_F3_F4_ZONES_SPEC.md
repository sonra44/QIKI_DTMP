# 04. F2 / F3 / F4 — спецификация

Статус: target spec.

## F2 «Подсистемы» — единый владелец порогов

- Проблема (дефект 0.17): локальные копии порогов расходятся с shared —
  `orion_v/modules/power.py:70-76` даёт 30%/24В против shared 20%/22В,
  поэтому чипы F1 и страницы F2 показывают разные статусы одного борта.
- Изменения:
  1. Снести локальные пороги: `modules/power.py:70-76`,
     `screens/cockpit.py:1866-1896`, `collector.py:2355`,
     `operator_state.py:1416` → импорт из владельца `qiki/shared`
     (продолжение «SelfModel Срез 0», коммит 9bc4a7c).
  2. `power_thermal_view_model.py`: NaN не проходит в термо-агрегаты;
     `0.0°C` — валидное измерение, а не «нет данных» (убрать falsy-or).
- Критерий: F1-чип и F2-страница не могут дать разные статусы по одному
  и тому же полю (приёмка §F2 в `07_ACCEPTANCE_CRITERIA.md`).

## F3 «Глубокий анализ» — изменений v1 НЕТ

Зафиксировано явно (анти-скоуп-дрейф): F3 наследует честность Блока 0
(инциденты/события/safe-mode уже рендерятся корректно, `screens/deep_dive.py`).

## F4 «Консоль» — командная поверхность с аффордансом

Текущий роутер: `app.py:1441-1569` (`on_input_submitted`). Изменения:

1. **`help` — полный сгруппированный список** typed-команд, сверенный с
   фактическим роутером (сейчас справка неполна). Группы: уровни (f1–f8),
   мир (`sim.start/pause/stop`), процедуры (`proc run …`), QIKI
   (`q: <текст>`, `q confirm`, `q cancel`, `q abort`), фильтры/страницы.
2. **`f5`/`f8` в переключателе уровней** (`app.py:1475`) — дефект 0.13.
3. **Голая `q` ≠ quit** (дефект 0.12): в командном режиме `q` без аргументов
   → подсказка (`q: <текст> — запрос QIKI; quit — выход`); выход из консоли —
   только `quit`/`exit` с подтверждением.
4. **Дублирование в палитре Ctrl+P** (`get_system_commands`): все ключевые
   typed-команды доступны из палитры — обнаруживаемость без чтения доков.

Критерий: новый оператор находит любую команду через `help` или палитру,
не читая исходников (приёмка §F4).
