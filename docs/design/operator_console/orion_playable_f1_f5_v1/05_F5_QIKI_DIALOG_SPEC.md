# 05. F5 «QIKI / Диалог» — дельта v1

Статус: target spec. База — утверждённый
`docs/design/operator_console/F5_QIKI_DIALOG_SYSTEM_DESIGN.md`
(CaMeL-граница; 4 сущности: диалог / кандидат / предпросмотр решения /
улика; лестница `intent → candidate → operator approve → CommandDecision →
тело`). Дизайн F5 НЕ пересматривается — здесь только дельта.

## Фиксы (из Блока 0, консольная часть)

1. **Полуночная сортировка** (дефект 0.14, `screens/qiki_dialog.py:94`):
   сортировать объединённую ленту по полному timestamp (дата+время),
   не по строке `HH:MM:SSZ`.
2. **Сброс pending** (дефект 0.15): `_pending_ack_command_id` сбрасывается
   после resolve/timeout; счётчик P в Action Rail отражает реальные ожидания.
3. **Таймауты awaiting**: висящий кандидат/ожидание ACK получает видимый
   таймаут-статус, не вечное «ждём» (соответствие «no silent failure»).

## Контекстный вход с F1

Выбор контекстного действия на F1 (Z6, `03_F1_COCKPIT_SPEC.md`) публикует
intent в `qiki.intents` — точно тот же путь, что ручной ввод `q: <текст>`:

- кандидат и предпросмотр появляются в штатном F5-контуре;
- F1 показывает краткий итог голосом QIKI (лента `QIKI ▸`), полная карточка —
  на F5;
- при активном кандидате подсказки `q confirm` / `q cancel` видимы и на F1
  (строка-подсказка), и на F5.

## Законы, которые дельта обязана соблюдать

- Никаких новых NATS-сабжектов и enum-значений без ADR (закон F5-дизайна §1.5).
- Провайдерский текст ≠ runtime-правда; LLM-выход не владеет control flow
  (CaMeL). Дельта не трогает генерацию — только консольный показ и фиксы.
- Неспрошенные ответы отклоняются (`app.py:4217`, `QIKI_RESP_UNSOLICITED`) —
  поведение сохраняется, тесты не ослабляются.
- m5-спуф-тесты (подмена пломбы) — неприкосновенны (handoff §4).

## LLM-зависимость

Голос Mercury требует локально настроенного gateway (Q3 в
`_support/CLARIFICATION_REQUEST_001.md`). Все приёмки F5 формулируются так,
чтобы политический (policy-only, без LLM) путь проходил их полностью;
LLM-реплики — дополнение, не условие приёмки.
