# 06. Командная поверхность и путь команды — центральное решение

Статус: target spec (ключевой архитектурный выбор пакета).

## Решение

**F1 никогда не публикует команд телу. Контекстные действия F1 публикуют
*intent* (`qiki.intents`). Исполнение достигает тела только через F5-контур:
кандидат → `q confirm` → пломба `CommandDecision` → authorize →
`decision_body_bridge`.**

## Обоснование

1. **Канон G1**: шаг цикла — «Запрос/Команда QIKI»; intent — это запрос,
   не команда. `MVP_NO_AUTO_ACTIONS` не нарушается: автоисполнения нет,
   кандидат неисполняем без подтверждения оператора (ступени propose/commit
   F5-дизайна §3).
2. **Пин-инвариант** «команды боту НЕ отправляются с F1»
   (`tests/unit/test_orion_f1_first_playable_loop.py`) **остаётся истинным
   буквально**: на control bus (`qiki.commands.control`) с F1 не уходит
   ничего; уходит только intent — как и при ручном `q: <текст>`. При правке
   пин-теста (этапы 5/9) старый инвариант заменяется эквивалентным новым
   тестом: «ни одно действие F1 не публикует в `qiki.commands.control`».
3. **Единственность пути пломбы** — вся защита (m5-спуф-тесты, TOCTOU-фиксы
   0.5) сосредоточена на одном пути; второй путь исполнения = второй контур
   защиты = недопустимая площадь атаки.

## Исключения

- **Мета-уровень `МИР` (`sim.start/pause/stop`)** — управление симуляцией,
  не командование телу; остаётся прямым `_publish_sim_command`
  (`app.py:1971`) с ACK-ожиданием. Это пауза игры, диегетике не принадлежит.
- **BRAKE OVERRIDE (гэп G4, «большой красный»)** — потенциально единственный
  кандидат на прямой аварийный путь мимо диалога. **В v1 не делается**;
  требует отдельного ADR (вопрос Q8 CLI-агенту).

## Карта пути команды (файлы)

```
F1 softkey / F5 ввод "q: …"
  → intent: QikiChatRequestV1/V2 → qiki.intents        (app.py:2195, _publish_qiki_intent)
  → q_core_agent: политика → ответ с кандидатом        (qiki_orion_intents_service.py)
  → консоль: кандидат + предпросмотр                    (app.py, _extract_qiki_pending_action)
  → оператор: q confirm                                 (роутер app.py:1441-1569)
  → пломба: CommandDecision.seal (deepcopy, digest)     (shared/command_decision.py)
  → авторизация против пломбы                           (app.py:3167, _authorize_pending_against_seal)
  → эффект: bridge → тело (пересверка digest)           (shared/decision_body_bridge.py)
  → ACK + телеметрия → Consequence Confirmation         (qiki.responses.control, qiki.telemetry)
```

## Карта клавиш: текущее → целевое

Текущие BINDINGS: `app.py:232-271`.

| Клавиша | Сейчас | Целевое v1 |
|---------|--------|-----------|
| F1–F8 | зоны | без изменений |
| `←/→` | prev/next учебного действия (F1) | prev/next контекстного действия сцены |
| `space`/`enter` | preview/apply учебного действия | preview/отправка intent выбранного действия |
| `q` (глобально) | quit | quit — только вне командного режима; в командном режиме см. F4-правило (04 §3) |
| `[` `]` | циклы MFD-страниц | без изменений |
| `h/H` | Help toggle | Help·OFF по умолчанию после первого применённого действия |
| `alt+1..6`, `a`, `x`, `e/E`, `b/B`, `n/p`, `/` `:` , `esc`, `pageup/pagedown` | как есть | без изменений |

## Карта typed-команд (роутер `app.py:1441-1569`)

Полный сгруппированный список — обязанность `help` (04 §F4-1). Изменения v1:
`f5`/`f8` добавляются в переключатель (`app.py:1475`); голая `q` перестаёт
быть quit (04 §F4-3); прочие команды роутера не меняются.
