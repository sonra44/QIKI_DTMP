## A8 — позиция оператора

**Да: отдельный ADR готовить. Но design-only и строго после Блока 0; в v1
не реализуется (дефолт пакета подтверждён).**

Рамки для будущего ADR (обязательные, из канона):

1. «Большой красный» минует **диалог/LLM-контур**, но НЕ минует
   **пломбу CommandDecision, legality-гейт и аудит**. Аварийность = короткий
   путь одобрения (одно действие оператора вместо цикла предложение→
   подтверждение), а не второй источник команд. Инвариант «одобрено =
   исполнено» сохраняется; dual-source решений запрещён
   (`CONTEXT_LOCK_QIKI_DTMP.md` п.5).
2. Safe Mode authority остаётся Q-Core Agent — override не создаёт
   конкурирующего органа безопасности.
3. Мотивация: в hardcore-пульте аварийный стоп обязателен по критерию
   операторской значимости; ожидание такого пути возникнет на этапе 9
   (контекстные действия Burn/Brake, G-D).
4. Триггер написания ADR: после закрытия Блока 0 (этапы 1–3), до старта
   этапа 9. До того — никакой реализации, только этот зафиксированный скоуп.
