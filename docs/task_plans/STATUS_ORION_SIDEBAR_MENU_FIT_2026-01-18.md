# STATUS: ORION — sidebar меню «не влезает» (2026-01-18)

## Симптом
В левом меню (sidebar) строки с хоткеями `Ctrl+N/Ctrl+P/Ctrl+T` обрезались через `…`:
- `Ctrl+N [Sensors/Сенс…]`
- `Ctrl+P [Propulsion/Д…]`
- `Ctrl+T [Thermal/Тепл…]`

Это выглядело как “текст не влезает” и портило читабельность.

## Причина
Sidebar выводил **длинные двуязычные заголовки** + декоративные скобки `[]`, при том что ширина sidebar в tmux ограничена.

## Исправление
- Sidebar теперь использует **короткие русские подписи без сокращений** (например: `Сенсоры`, `Двигатели`, `Тепло`).
- Формат строки в sidebar упрощён: `▶ F1 Система`, `Ctrl+P Двигатели` (без `[]`), чтобы не тратить ширину на декор.

## Инварианты
- no-mocks: данные не трогаем, только UI.
- no-v2 / no-duplicates.

## Проверка
- Docker tests: `pytest -q src/qiki/services/operator_console/tests` — зелёный.
- В tmux: строки меню больше не режутся, хоткеи читаемы.

## Изменённый файл
- `src/qiki/services/operator_console/main_orion.py` (`OrionSidebar.render` + новые константы лейблов)
