# WORKLOG — ORION Sensors: compact view + toggle — 2026-01-18

## Проблема

Экран `Sensors/Сенс` занимал слишком много места: построчная детализация (14+ строк) превращала экран в “простыню” и мешала
ориентироваться, особенно когда появится ещё больше виртуализируемых параметров.

## Цель

Сделать Sensors экран **сводным** (glanceable), а детали — доступными **по требованию** (скрыть/раскрыть) без моков.

## Инварианты

- **no-mocks**: если данных нет → `N/A`, не рисуем “OK” из воздуха.
- **no `*_v2` / no duplicates**: не создаём параллельные каноны/схемы.
- **Docker-first**: тесты/проверки — в Docker.

## Что изменено

Файл:
- `src/qiki/services/operator_console/main_orion.py`

Изменения:

1) **Sensors по умолчанию стал компактным**
- Вместо “каждый параметр отдельной строкой” показываем 5–6 строк по подсистемам:
  - `IMU`, `Radiation`, `Proximity`, `Solar`, `Star tracker`, `Magnetometer`.
- В колонке `Value/Значение` — ключевой показатель (без имитации).
- Колонки `Age` и `Source` в compact-режиме пустые (чтобы не сжигать ширину и не забивать повторяющимся `telemetry/телеметрия`).

2) **Раскрытие/скрытие**
- На таблице Sensors `Enter` переключает **compact ↔ details** (через обработчик `on_data_table_row_selected`).
- В keybar добавлена подсказка только для Sensors: `Enter — компакт/детали`.

3) **Доступ к деталям без “простыни”**
- Даже в compact-режиме детали доступны через **Inspector** (у нас уже увеличены лимиты preview в предыдущем круге).

## Проверка

- `pytest -q src/qiki/services/operator_console/tests` — зелёный (в Docker).
- В runtime проверено визуально: Sensors показывает компактный список.

## Заметки / что можно улучшить дальше

- Для `Proximity/Solar/…` можно выводить “enabled yes/no” как ключевую метрику, чтобы меньше было `N/A` в Status.
- Если захотим “аккордеон” (раскрытие группы внутри одной таблицы) — можно добавить позже, но текущий toggle уже закрывает боль.
